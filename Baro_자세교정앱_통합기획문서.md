## 1. 앱 컨셉

### 1.1 문제 정의
현대인은 장시간 좌식 생활로 거북목·라운드숄더가 일상화됐지만, 집중 상태에선 자세 인지가 끊기고 환경적 제약 탓에 교정이 어렵다. 기존 자세교정 앱은 사용자의 의지에 의존하고, 기계적·일방적 알림으로 작업 흐름을 끊어 '잔소리'처럼 느껴지기 때문에 대부분 단기간에 중단된다.

### 1.2 솔루션 방향
의지에 기대지 않고 작업 흐름을 방해하지 않는 **초저관여 개입**이 필요하다.
- **마이크로넛지** — 자세를 자연스럽게 인지시킨다.
- **게이미피케이션 보상** — 캐릭터 성장·연속 일수·챌린지로 지속을 유도한다.

### 1.3 핵심 컨셉 요약
| 항목 | 내용 |
| --- | --- |
| 앱 이름 | **Baro (바로)** |
| 마스코트 | 거북이 캐릭터 — 스트레칭을 지속할수록 `인간화` 단계로 성장 (예: 인간화 2단계) |
| 컨셉 내러티브 | '거북목 → 거북이'를 의인화. 자세가 교정될수록 거북이가 사람에 가까워지며 '거북이 탈출' |
| 핵심 가치 | 흐름을 끊지 않는 자세 인지 + 지속 가능한 보상 |
| 주요 기능 | 스트레칭 추천·알림 / 기록·분석 / 친구·챌린지 / 캐릭터 성장 |

## 2. 정보구조도 (IA)

### 2.1 진입 & 내비게이션
- **진입** : 스플래쉬 → 로그인
- **하단 탭바 (4)** : `A 홈화면` · `B 나의 기록` · `C 친구와 함께` · `D 알림 설정`
- **상단 바** : `E 마이페이지`(프로필 아이콘) · `F 어플 알림 확인`(벨 아이콘)

### 2.2 화면 구조

**A. 홈화면** — 언제 스트레칭을 실행했는지 캘린더를 통해 지금까지의 기록을 보여준다.
- **A-1 내 스트레칭** — 당일 실행 여부 / 일주일 간 실행 여부 / 누적 스트레칭 일수 / 동기부여 멘트 / 나의 목표
- **A-2 오늘의 자세** — 목표 설정·수정 / 오늘의 스트레칭 정보 / 스트레칭 챌린지 / 스트레칭 수행 횟수
- **A-3 스트레칭 알림** — 다음 실행까지 남은 시간 / 현재 상태 시각화 아이콘 / 알림 미루기 / 알림 끄기

**B. 나의 기록** — 스트레칭 실행 기록을 분석해 내 자세의 심각도와 유형을 알려준다.
- **B-1 상태 시각화** — 자세 교정 정도를 시각화하여 현재 얼마나 했는지, 어떤 단계인지 보여준다.
- **B-2 캘린더** — 스트레칭을 언제, 얼마나 실행했는지 확인한다.
- **B-3 내 자세 분석** — 스트레칭 강도·횟수·시간, 카메라 인식 결과를 분석해 스트레칭 유형을 알려준다.

**C. 친구와 함께** — 친구 유저 목록과 당일 스트레칭 여부·실행 시간을 확인한다.
- **C-1 친구 리스트** — 친구의 실행 여부 / 이모티콘 보내기 / 지정 문구 보내기 / 같이 스트레칭 요청 / 친구 추가(링크 복사·카카오톡·DM·X·SMS)
- **C-2 스트레칭 챌린지** — 친구가 오늘 실행한 스트레칭 정보 / 나만의 스트레칭 자세 공유 / 유저들이 많이 하는 스트레칭 자세

**D. 알림 설정** — 스트레칭 알림의 시간·유형을 설정하고 수정한다.
- **D-1 스트레칭 알림 표시** — 다음 알림까지 남은 시간 / 현재 상태 시각화 아이콘 / 알림 미루기 / 알림 끄기
- **D-2 알림 설정 수정** — 요일 설정 / 허용 시간 / 거부 시간 / 빈도 설정 / 유형 설정(유·무음·진동·문구형·풀스크린형·작은 아이콘형)

**E. 마이페이지** — 프로필·목표·앱 설정을 확인·수정한다.
- **E-1 프로필** — 상태 시각화 아이콘 / 이름(닉네임) / 스트레칭 목표 / 나이 / 키 / 몸무게 / 겪고 있는 통증
- **E-2 프로필 수정** — 이름 / 목표 / 생활 습관 / 연동 계정 / 연동 연락처 / 회원 탈퇴
- **E-3 앱 설정 변경** — 카메라 권한 / 위치 권한 / 주소록 권한 / 메일 알림 / SMS 알림 / 친구 알림

**F. 어플 알림 확인** — 어플에서 온 각종 알림을 확인한다.

## 3. 화면정의서

> 상세 UI 시안이 제공된 화면은 **A 홈화면**과 **B 나의 기록**이며, C~F는 IA 기준 구성 요소로 정리한다.

### 3.1 A_홈화면
**화면 위치** · 홈화면 (진입 기본 화면)

| No. | 요소 | 설명 |
| --- | --- | --- |
| 1 | 타이틀 | 어플명·카테고리명을 명시해 현재 페이지를 설명 |
| 2 | 마이페이지 | 아이콘 터치 시 `E 마이페이지`로 이동 |
| 3 | 연속 스트레칭 일수 | 당일 기준, 연속 며칠 스트레칭했는지 표시 |
| 4 | 사용자의 목표 | 회원가입 시 입력한 목표 표시 |
| 4-1 | 목표 수정 버튼 | 설정한 목표를 수정 (기본 상태 → 수정 가능 상태로 전환) |
| 5 | 내 스트레칭 | 대표 캐릭터·동기부여 문장·오늘 스트레칭 횟수·주간 실행 여부 시각화 |
| 5-1 | 주간 스트레칭 실행 여부 | 월~일 요일별 실행 여부 표시 |
| 6 | 오늘의 스트레칭 | 오늘 추천 스트레칭을 여러 개 노출 (캐러셀, 예: 1/3) |
| 7 | 스트레칭 알림 | 다음 알림까지 남은 시간 표시 (미루기·끄기) |
| 8 | 챌린지 목록 | 유저·친구·개인이 진행 중인 챌린지 목록 |
| 9 | 홈 탭 | 터치 시 `A 홈화면` 이동 |
| 10 | 나의 기록 탭 | 터치 시 `B 나의 기록` 이동 |
| 11 | 친구 탭 | 터치 시 `C 친구와 함께` 이동 |
| 12 | 알림 탭 | 터치 시 알림 화면 이동 (※ 5장 정합성 메모 참고) |

**예시 콘텐츠 (시안 기준)**
- 연속 일수 · `🔥 23일차`
- 목표 문구 · `이번엔 진짜 한다!`
- 메인 카피 · `지구인님, 오늘도 자세를 바르게 유지해봐요!`
- 오늘의 스트레칭(1/3) · `굽은 등 바로 하기`(50초) · `틀어진 허리 바로 하기`(50초)
- 알림 타이머 · `01:12`
- 챌린지 · `D+32 거북이 탈출`(16명·32%) · `D+19 부기부기모임`(4명·45%)

## 4. 유저플로우 (태스크 플로우)

### 4.1 메인 태스크 — 나의 스트레칭 기록 확인
**시나리오** · 유저가 지난 달 스트레칭 여부를 한 번에 확인하고 싶어한다.

```
시작
  → A 홈화면
  → 하단 탭바에서 '나의 기록' 선택 (B 나의 기록 이동)
  → 캘린더 확인
  → 캘린더 월 이동 버튼(◀)으로 지난 달 기록 확인
종료
```

| 단계 | 화면 | 행동 |
| --- | --- | --- |
| 1 | A 홈화면 | 앱 진입 |
| 2 | B 나의 기록 | 하단 탭에서 '나의 기록' 이동 |
| 3 | B-2 캘린더 | 월간 캘린더에서 실행 일자 확인 |
| 4 | B-2 캘린더 | 월 이동 버튼으로 지난 달 전환 → 기록 확인 |

> 위 플로우는 제공된 태스크 플로우 문서의 대표 시나리오다. 스트레칭 실행, 알림 미루기, 친구 추가, 챌린지 참여 등 보조 플로우는 2장 IA와 3장 화면정의서 구조를 따른다.

## 5. 참고 (정합성 메모)
- 하단 '알림' 탭의 이동 대상이 시안마다 다르게 표기됨 (`A 홈화면` 시안 → E_알림창, `B 나의 기록` 시안 → D_알림창). IA의 `D 알림 설정`을 기준으로 통일 권장.
- 마이페이지 하위에 `E-2`가 두 번(프로필 수정 / 앱 설정 변경) 표기됨 → 앱 설정 변경을 `E-3`으로 정리함.
- 상단 벨 아이콘의 이동 대상(`F 어플 알림 확인`)과 하단 알림 탭(`D 알림 설정`)의 역할 구분을 화면정의서에 명시 권장.
